期望 DP

前置知识：概率与期望。

期望 DP ，通常来说就是求某个东西的期望值。一般来说，我们可以知道终态的期望值，通过终态逆推得到结果。

解决过程

对于期望 DP ，我们一般需要找到这两个要点。

  • 能够确定的状态是什么？（一般来说，是终态）
  • 状态转移方程是什么？

我们为什么要找终态呢？因为，对于期望问题来说，我们一开始能够确定的，就是问题的终态。例如，从 111 楼随机坐电梯到 nnn 楼，到达 nnn 楼的期望次数是多少。显然，这个问题中的终态就是 E(n)E(n)E(n) ，也就是 nnn 楼，其期望步数 E(n)=0E(n) = 0E(n)=0 ，我们可以通过这个已知的状态，去逆推到 E(1)E(1)E(1) 的状态。

至于状态转移方程，这是很多 DP 问题中需要思考的一个问题。在期望问题中，状态转移方程需要基于期望值的计算公式（可以参考一下之前写过的概率与期望篇章），要通过计算期望的式子才可以正确得到状态转移方程。但一般来说，对于期望 DP 问题，我们使用一些公式加上题目所暗示的状态转移方式，得到的方程出现了左右两边都有我们需要求得的状态（这也是大多 DP 会遇到的问题），这时候我们就需要进行额外化简，才能确保我们的状态转移方程是正确的。

例题 LightOJ-1030 Discovering Gold

题目传送门

题目简述

现在有一个长度为 nnn 的黄金走廊，你可以把他理解为一个 1⋅n1 \cdot n1⋅n 的序列。每一个位置都可以含有一定数量的黄金。最初你处于位置 111。现在每个回合你掷一个 666 面骰子。假设掷出的数字为 XXX，这时你的位置就会加上 XXX，然后从新的位置收集黄金。如果当前位置加上 XXX 后的位置超出了 nnn ，则重新掷直到不超过 nnn 。当你到达第 nnn 个位置时，你就停止了你的旅程。现在已知序列，请你计算你可以收集到的黄金的预期数量。

思路

我们考虑这道问题的终态是什么？到达第 nnn 个位置时将会停下。但是我们不知道 E(到达n时黄金数量)E(到达n时黄金数量)E(到达n时黄金数量) 是多少啊？不过我们可以转换一个思路：将”到达 nnn 时黄金数量“转换为”从 xxx 到达 nnn 时的黄金数量“，这也是问题所要问的从 111 到 nnn 的期望黄金数量。因此，现在我们的期望值变成了 E(从x到达n时的黄金数量)E(从x到达n时的黄金数量)E(从x到达n时的黄金数量) ，那么从 nnn 处出发到达 nnn 处的黄金期望数量自然是第 nnn 处的黄金数量了。

那么我们考虑一下状态转移是如何转移的？当然是所有能够到达的位置的期望黄金*可以到达那个位置的概率
之和了（考虑求期望的式子），这样，我们只需要输出 E(1)E(1)E(1) ，就是这道题的答案了。

#include <bits/stdc++.h>
using namespace std;

int main() {
	std::cin.tie(nullptr)->sync_with_stdio(false);
	std::cout << std::fixed << std::setprecision(14);
	int t;
	cin >> t;
	for (int i = 1; i <= t; i ++) {
		cout << "Case " << i << ": ";
		int n;
		cin >> n;
		vector<double> f(n);
		for (auto &i : f) {
			cin >> i;
		}
		for (int i = n - 1; i >= 0; i --) {
			for (int j = 1, k = min(n - 1 - i, 6); j <= k; j ++) {
				f[i] += f[i + j] / k;
			}
		}
		cout << f[0] << '\n';
	}
}

例题 LightOJ-1038 Race to 1 Again

题目传送门

题目简述

给定一个整数 nnn ，每次操作可以当前数进行除以它的某个因子。除以某个因子是随机的，求把 nnn 变成 111 的期望步数。

思路

我们对于分析期望 DP ，一般要找到这两个步骤：

  • 终态是什么？
  • 状态转移方程是什么？

有了终态和状态转移方程，我们才可以从终态逆着转移，转移到我们要求的位置。状态转移的方法很多，可以是记忆化搜索，也可以是直接循环。

回到这道题，我们可以发现终态是 111 ，111 的因子是它自身，同时也达到它自身，因此 E(1)=0E(1) = 0E(1)=0 成立。

接着，我们设我们要找的答案为 E(x)E(x)E(x) ，对于 xxx 的 kkk 个约数按从小到大的顺序记为 d1,d2,⋯ ,dkd_1, d_2, \cdots , d_kd1​,d2​,⋯,dk​ 。因此，我们可以简单地按照 EX=∑ω∈ΩX(ω)Pr⁡(Ω)EX = \sum_{\omega \in \Omega} X(\omega) \operatorname{Pr}(\Omega)EX=∑ω∈Ω​X(ω)Pr(Ω) 的式子，求得 E(x)E(x)E(x) 的值。

E(x)=(E(d1)+1)⋅1k+(E(d2)1)⋅1k+⋅(E(dk−1+1))⋅1k+⋅(E(dk+1))⋅1kE(x) = (E(d_1) + 1) \cdot \cfrac{1}{k} + (E(d_2) _ 1) \cdot \cfrac{1}{k} + \cdot (E(d_{k-1} + 1)) \cdot \cfrac{1}{k} + \cdot (E(d_{k} + 1)) \cdot \cfrac{1}{k} E(x)=(E(d1​)+1)⋅k1​+(E(d2​)1​)⋅k1​+⋅(E(dk−1​+1))⋅k1​+⋅(E(dk​+1))⋅k1​

很明显的，我们得到了一个状态转移方程。我们不妨想一下：d1,dkd_1, d_kd1​,dk​ 是什么？显然是 d1=1,dk=xd_1 = 1, d_k = xd1​=1,dk​=x 。因此，我们现在又获得一个新的状态转移方程。

对上述式子化简，可以得到这样的式子。

E(x)=(E(1)+E(d2)+⋯+E(dk−1)+k)k−1E(x) = \cfrac{(E(1) + E(d_2) + \cdots + E(d_{k - 1}) + k)}{k-1} E(x)=k−1(E(1)+E(d2​)+⋯+E(dk−1​)+k)​

因此，预处理递推一下就好了。

#include <bits/stdc++.h>
using namespace std;

const int N = 1e5 + 10;
double f[N];

int main() {
	std::cin.tie(nullptr)->sync_with_stdio(false);
	std::cout << std::fixed << std::setprecision(12);
	f[0] = f[1] = 0;
	for (int i = 2; i <= 1e5; i ++) {
		int cnt{}; double tot{};
		for (int j = 1; j * j <= i; j ++) {
			if (i % j == 0) {
				tot += f[j] + 1;
				cnt ++;
				if (j * j != i) {
					tot += f[i / j] + 1;
					cnt ++;
				}
			}
		}
		f[i] = tot / (cnt - 1);
	}
	int t;
	cin >> t;
	for (int i = 1; i <= t; i ++) {
		int x;
		cin >> x;
		cout << "Case " << i << ": " << f[x] << '\n';
	}
}

例题 P4316 绿豆蛙的归宿

题目传送门

题目标签

DAG 上 DP + 期望

题目描述

给出张 nnn 个点 mmm 条边的有向无环图，起点为 111，终点为 nnn，每条边都有一个长度，并且从起点出发能够到达所有的点，所有的点也都能够到达终点。

绿豆蛙从起点出发，走向终点。 到达每一个顶点时，如果该节点有 kkk 条出边，绿豆蛙可以选择任意一条边离开该点，并且走向每条边的概率为 1k\frac{1}{k}k1​ 。现在绿豆蛙想知道，从起点走到终点的所经过的路径总长度期望是多少？

思路

我们首先可以尝试定义 E(x)E(x)E(x) ，也就是从 111 到达 xxx 的期望长度，但是发现很难转移，因此转换思路，为从 xxx 到 nnn 的路径期望长度，那么终态 E(n)=0E(n) = 0E(n)=0 ，然后状态转移式子就很容易得到了。

E(v)=∑u→vE(u)+w(u,v)indegree(v)E(v) = \sum_{u \to v} \cfrac{E(u) + w(u, v)}{indegree(v)} E(v)=u→v∑​indegree(v)E(u)+w(u,v)​

对原图做反图后，从 nnn 开始做拓扑排序（或者正常的 bfs），最后答案自然是 E(1)E(1)E(1) ，表示从 111 到 nnn 的期望路径长度。

#include <bits/stdc++.h>
using namespace std;

const int N = 1e5 + 10;
vector<pair<int,double>> edge[N];
double f[N];
int ind[N], iind[N];
void add_edge(int from, int to, double w) {
	edge[from].emplace_back(to, w);
	edge[to].emplace_back(from, w);
}

int main() {
	std::cin.tie(nullptr)->sync_with_stdio(false);
	std::cout << std::fixed << std::setprecision(2);
	int n, m;
	cin >> n >> m;
	for (int i = 1; i <= m; i ++) {
		int a, b;
		double w;
		cin >> a >> b >> w;
		add_edge(b, a, w);
		ind[a] ++;
		iind[a] ++;
	}
	queue<int> q;
	q.push(n);
	while (!q.empty()) {
		int from = q.front(); q.pop();
		for (auto [to, w] : edge[from]) {
			f[to] += (f[from] + w) / iind[to];
			if (-- ind[to] == 0) {
				q.push(to);
			}
		}
	}
	cout << f[1] << '\n';
}
